NeuroLint CLI
npm version npm downloads License: MIT

TL;DR: One command modernizes React/Next.js codebases safely (a11y, SSR, React 19, Next.js 15.5) with dry-runs, backups, and validated transforms.

The Problem: Modern React/Next.js development is plagued with repetitive code issues - missing accessibility attributes, hydration errors, outdated patterns, and inconsistent configurations. Manual fixes are time-consuming and error-prone.

The Solution: NeuroLint CLI automatically detects and fixes 50+ common issues across 7 intelligent layers, transforming your codebase from legacy patterns to modern best practices in minutes.

Quick Start

npm install -g @neurolint/cli

# Analyze first (no writes)
neurolint analyze ./src --verbose

# Preview changes (no writes)
neurolint fix ./src --dry-run --layers 1,2,3,4,5 --verbose

# Apply with backups
neurolint fix ./src --layers 1,2,3,4,5 --verbose
Demo (GIF): See short CLI demo at https://neurolint.dev

'use client';

import React from 'react';

interface ButtonProps {
  children: React.ReactNode;
  onClick?: () => void;
  variant?: 'primary' | 'secondary';
  disabled?: boolean;
}

function Button({ 
  children, 
  onClick, 
  variant = 'primary',
  disabled = false 
}: ButtonProps) {
  return (
    <button 
      onClick={onClick}
      disabled={disabled}
      className={`btn btn-${variant}`}
      aria-label={typeof children === 'string' ? children : undefined}
    >
      {children}
    </button>
  );
}

export default Button;

Layer 4: SSR/Hydration Safety
Problem: Client-side APIs causing hydration errors

// Before
const Component = () => {
  const [theme, setTheme] = useState(localStorage.getItem('theme'));
  return <div>{window.innerWidth}px</div>;
}

// After
const Component = () => {
  const [theme, setTheme] = useState<string | null>(null);
  
  useEffect(() => {
    setTheme(localStorage.getItem('theme'));
  }, []);

  const [width, setWidth] = useState<number>(0);
  
  useEffect(() => {
    setWidth(window.innerWidth);
  }, []);

  return <div>{width}px</div>;
}

Addressing Next.js Complexity Concerns
NeuroLint directly addresses the r/nextjs community discussion about Next.js complexity for small projects:

"Is Next.js 15 getting too complicated for small projects?"
The Problem: As discussed on Reddit, Next.js introduces many concepts (server components, app router, middleware) that can feel overwhelming for simple projects.

NeuroLint's Solution: Smart complexity management that scales with your needs.

When to Use What
Use Plain React When:
Building simple SPAs or static sites
No SSR/SSG requirements
Team prefers simplicity over features
Learning React fundamentals
# Convert your Next.js project to plain React
neurolint assess . --verbose          # Analyze current complexity
neurolint simplify . --target=react   # Convert to React SPA
Use Minimal Next.js When:
Need basic routing and build optimization
Want to keep upgrade path for future features
Team is comfortable with Next.js
# Remove unnecessary complexity while keeping Next.js
neurolint simplify . --target=minimal-nextjs --dry-run
Keep Full Next.js When:
Using SSR, SSG, or API routes
Building production applications
Need advanced performance optimizations
# Just clean up and modernize existing setup
neurolint fix . --layers=1,2,5 --dry-run

Issue: neurolint migrate-nextjs-15.5 command not found Solution: Ensure you are on v1.4.0+ and use one of:

# Dedicated Next.js 15.5 migration
neurolint migrate-nextjs-15.5 . --dry-run --verbose

neurolint migrate . --layers=1,2,5 --dry-run --verbose

Backup System
NeuroLint includes a comprehensive backup system with both basic and enterprise-grade options.

Basic Backup System (Default)
Location: .neurolint-backups/ directory in your project root
Format: JSON files with metadata and content
Safety: MD5 hash verification and integrity checks
Retention: Keeps latest 10 backups per file
Enterprise Backup System (Production Mode)
Activate with --production flag for enterprise-grade features:

Encryption: AES-256-GCM encryption with automatic key rotation
Compression: Built-in compression (typically 30-50% reduction)
Monitoring: Real-time metrics, health checks, and alerting
Security: Role-based access control and audit trails
Verification: Automatic backup integrity verification
Disaster Recovery: Automated recovery procedures
Retention: Configurable retention (default: 50 backups)
Structured Logging: JSON logs with correlation IDs for troubleshooting
